\doxysubsection{Detailed Description}
Handle to an Azure Kinect capture. 

\begin{DoxyRemark}{Remarks}
A capture represents a set of images that were captured by a device at approximately the same time. A capture may have a color, IR, and depth image. A capture may have up to one image of each type. A capture may have no image for a given type as well.

Captures also store a temperature value which represents the temperature of the device at the time of the capture.

While all the images associated with the capture were collected at approximately the same time, each image has an individual timestamp which may differ from each other. If the device was configured to capture depth and color images separated by a delay, \mbox{\hyperlink{group___functions_ga4dac757a33657f4d3dbf1ae8b21c158a_ga4dac757a33657f4d3dbf1ae8b21c158a}{k4a\+\_\+device\+\_\+get\+\_\+capture()}} will return a capture containing both image types separated by the configured delay.

The capture will add a reference on any \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} that is added to it with one of the setters like \mbox{\hyperlink{group___functions_ga28824350a175d8eeba7c7a7d2753c2cf_ga28824350a175d8eeba7c7a7d2753c2cf}{k4a\+\_\+capture\+\_\+set\+\_\+color\+\_\+image()}}. If an existing image is being replaced, the previous image will have the reference released.

Once all references to the \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} are released, the capture will release the references it holds on any \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} that are associated with it.

When a capture returns an image to the caller with an accessor like \mbox{\hyperlink{group___functions_ga683e440b5f22215a2de58d7fa140488c_ga683e440b5f22215a2de58d7fa140488c}{k4a\+\_\+capture\+\_\+get\+\_\+color\+\_\+image()}}, a new reference is added to the \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} and the caller is responsible for releasing that reference with \mbox{\hyperlink{group___functions_ga27c81863b13fafc3934a32935a014e9f_ga27c81863b13fafc3934a32935a014e9f}{k4a\+\_\+image\+\_\+release()}}.

Empty captures are created with \mbox{\hyperlink{group___functions_gaffbf32b4d014c31826d52de1c0c932cc_gaffbf32b4d014c31826d52de1c0c932cc}{k4a\+\_\+capture\+\_\+create()}}.

Captures can be obtained from a device using \mbox{\hyperlink{group___functions_ga4dac757a33657f4d3dbf1ae8b21c158a_ga4dac757a33657f4d3dbf1ae8b21c158a}{k4a\+\_\+device\+\_\+get\+\_\+capture()}}.

The caller must call \mbox{\hyperlink{group___functions_ga0ed6f74ec403c3eac1b8ef3afb32cee6_ga0ed6f74ec403c3eac1b8ef3afb32cee6}{k4a\+\_\+capture\+\_\+release()}} to release its reference to any \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} that it receives from an Azure Kinect API.

Additional references to the capture may be obtained with \mbox{\hyperlink{group___functions_gab15293d2e452470fff8d5109472c7350_gab15293d2e452470fff8d5109472c7350}{k4a\+\_\+capture\+\_\+reference()}}.

An invalid \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} will be set to 0.
\end{DoxyRemark}
