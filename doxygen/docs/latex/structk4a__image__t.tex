\doxysubsection{Detailed Description}
Handle to an Azure Kinect image. 

\begin{DoxyRemark}{Remarks}
A \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} manages an image buffer and associated metadata.

The memory associated with the image buffer in an \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} may have been allocated by the Azure Kinect APIs or by the application. If the \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} was created by an Azure Kinect API, its memory will be freed when all references to the \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} are released. All images retrieved directly from a device will have been created by the API. An application can create a \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} using memory that it has allocated using \mbox{\hyperlink{group___functions_gaf84f2a271bcf6afae429bbccd47071b3_gaf84f2a271bcf6afae429bbccd47071b3}{k4a\+\_\+image\+\_\+create\+\_\+from\+\_\+buffer()}}. In that case the memory is not freed when the references are released but instead the application is provided with a callback. Applications need to create a \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} to provide as an output to the \mbox{\hyperlink{structk4a__transformation__t}{k4a\+\_\+transformation\+\_\+t}} transformation functions.

An image has a number of metadata properties that can be set or retrieved using the accessor functions. Not all properties are applicable to images of all types. See the documentation for the individual accessors for more information on their applicability and meaning.

Images may be of one of the standard \mbox{\hyperlink{group___enumerations_gabd9688eb20d5cb878fd22d36de882ddb_gabd9688eb20d5cb878fd22d36de882ddb}{k4a\+\_\+image\+\_\+format\+\_\+t}} formats, or may be of format \mbox{\hyperlink{group___enumerations_gabd9688eb20d5cb878fd22d36de882ddb_ggabd9688eb20d5cb878fd22d36de882ddba10833e30171730f43eeff1dc8d7b0a52}{K4\+A\+\_\+\+IMAGE\+\_\+\+FORMAT\+\_\+\+CUSTOM}}. The format defines how the underlying image buffer should be interpreted.

Images from a device are retrieved through a \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} object returned by \mbox{\hyperlink{group___functions_ga4dac757a33657f4d3dbf1ae8b21c158a_ga4dac757a33657f4d3dbf1ae8b21c158a}{k4a\+\_\+device\+\_\+get\+\_\+capture()}}.

Images stored in a \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} are referenced by the \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} until they are replaced or the \mbox{\hyperlink{structk4a__capture__t}{k4a\+\_\+capture\+\_\+t}} is destroyed.

The caller must call \mbox{\hyperlink{group___functions_ga27c81863b13fafc3934a32935a014e9f_ga27c81863b13fafc3934a32935a014e9f}{k4a\+\_\+image\+\_\+release()}} to release its reference to any \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} that it receives from an Azure Kinect API.

Additional references to the image may be obtained with \mbox{\hyperlink{group___functions_gae211d6b97d4ce1c752e8db9d5ea5b8b9_gae211d6b97d4ce1c752e8db9d5ea5b8b9}{k4a\+\_\+image\+\_\+reference()}}.

An invalid \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} will be set to 0.
\end{DoxyRemark}

\begin{DoxyRemark}{Remarks}
This function is used to create images of formats that have consistent stride. The function is not suitable for compressed formats that may not be represented by the same number of bytes per line.

For most image formats, the function will allocate an image buffer of size {\ttfamily height\+\_\+pixels} $\ast$ {\ttfamily stride\+\_\+bytes}. Buffers \mbox{\hyperlink{group___enumerations_gabd9688eb20d5cb878fd22d36de882ddb_ggabd9688eb20d5cb878fd22d36de882ddbaf334f6c32b6b44d1f5df02a349f05288}{K4\+A\+\_\+\+IMAGE\+\_\+\+FORMAT\+\_\+\+COLOR\+\_\+\+NV12}} format will allocate an additional {\ttfamily height\+\_\+pixels} / 2 set of lines (each of {\ttfamily stride\+\_\+bytes}). This function cannot be used to allocate \mbox{\hyperlink{group___enumerations_gabd9688eb20d5cb878fd22d36de882ddb_ggabd9688eb20d5cb878fd22d36de882ddbaae172e7eecc6f4adfc318c789e49aaf9}{K4\+A\+\_\+\+IMAGE\+\_\+\+FORMAT\+\_\+\+COLOR\+\_\+\+MJPG}} buffers.

To create an image object without the API allocating memory, or to represent an image that has a non-\/deterministic stride, use \mbox{\hyperlink{group___functions_gaf84f2a271bcf6afae429bbccd47071b3_gaf84f2a271bcf6afae429bbccd47071b3}{k4a\+\_\+image\+\_\+create\+\_\+from\+\_\+buffer()}}.

The \mbox{\hyperlink{structk4a__image__t}{k4a\+\_\+image\+\_\+t}} is created with a reference count of 1.

When finished using the created image, release it with k4a\+\_\+image\+\_\+release.
\end{DoxyRemark}
